## The Mock Listings Dataset

`data/listings.json` contains 40 mock secondhand listings across categories (tops, bottoms, outerwear, shoes, accessories) and styles (vintage, y2k, grunge, cottagecore, streetwear, and more).

Each listing has: `id`, `title`, `description`, `category`, `style_tags`, `size`, `condition`, `price`, `colors`, `brand`, and `platform`.

Load it with:
```python
from utils.data_loader import load_listings
listings = load_listings()
```

## The Wardrobe Schema

`data/wardrobe_schema.json` defines the format your agent uses to represent a user's existing wardrobe. It includes:

- `schema`: field definitions for a wardrobe item
- `example_wardrobe`: a sample wardrobe with 10 items you can use for testing
- `empty_wardrobe`: a starting template for a new user

Load an example wardrobe with:
```python
from utils.data_loader import get_example_wardrobe
wardrobe = get_example_wardrobe()
```

## Tool inventory

1) Search_listings
- Purpose: Searches the mock dataset and filters by user constraints
- Inputs:  Description (str): Keywords to match against title, description, and tags.
           Size (str)       : Exact case-insensitive substring match
           Max_price(Float) : Upper limit budget constraint
- outputs: list[dict], a sorted list of matched listing dictionaries, with (id, title, price, platform, etc)

2. suggest_outfit
- Purpose: Uses the Groq LLM to generate styling advice combining the new item with the user's existing closet.
- Inputs:  new_item (dict): The dictionary representation of the item found in Tool 1.
           wardrobe (dict): The user's wardrobe dictionary containing their existing clothes.
- Outputs: str (A paragraph of specific styling advice).

3. create_fit_card
- Purpose: Generates a short, emoji-filled social media caption for the outfit.
- Inputs: outfit (str): The styling suggestion generated by Tool 2.
          new_item (dict): The metadata of the secondhand piece (for price/platform bragging).
- Outputs: str (An Instagram/TikTok style caption).


## Planning loop and architecture

1) initialization: The agent extracts the search keywords, size, and price limits from the user's query text using an initial regex or explicit values.

2) Call search_listings(description, size, max_price).
     Branch A(Empty Path): if the return value is an empty list [], the agent sets session["error"] = "No items matched your description...", sets the selected item = none, and exits early.
     Branch B(Success Path): If items are found, set session["selected_item"] = results[0] (the top result), and continue.

3)  Outfit Styling: Inspect session["selected_item"]. if present, pass it along with the user's wardrobe array to suggest_outfit(). Save the returned text to session["outfit_suggestion"]

4) Caption Generation: Check if session["outfit_suggestion"] exists and isn't an error message. If valid, pass both the suggestion and session["selected_item"] to create_fit_card() and store the output string in session["fit_card"].

5) Termination: The loop is complete once session["fit_card"] is populated or when a branch sets session["error"] and exits early.

## State Management
- Information flows between tools using a single, centralized dictionary called session. Initialized by a _new_session() helper, this dictionary acts as the single source of truth for the entire run.
- The loop stores outputs in specific keys (session["selected_item"],session["outfit_suggestion"], etc ). When the run concludes, app.py maps these session keys to the gradio interface panels.

# Error Handling
Every tool has a failure sate to prevent the pipeline from breaking down.
1) search listings (Empty Search): if no results are found, it safely returns an empty list[]. The planning loop catches this and populates session["error"].
    - Example: Searching for "designer ballgown size XXS under $5" cleanly returns the message "No items matched your search. Try broadening your description, adjusting the size, or raising the price limit." It sends this error to the UI instead of breaking.
2) Suggest_outfit (Empty Wardrobe): If the user's wardrobe array is empty, the tool detects it and swaps to a secondary LLM prompt asking for "general styling guidance" instead of trying to match with non-existent clothes.
3) create_fit_card (Incomplete Input): A guard clause at the top of the function checks if the outfit string is empty. If it is, it bypasses the LLM entirely and returns a hardcoded error string, saving an API call and preventing a hallucinated response.
